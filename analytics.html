Analytics¶

To calculate analytics on a financial instrument, security, or smart contract, you may use the FinXClient client or the FinXRest client, depending on the function called.

The FinXClient is instantiated as follows:

Instantiate FinXClient¶
 import os
 import finx
 from finx.client import FinXClient
 finx_api_key = os.environ[FINX_API_KEY]
 if finx_api_key:
     finx_client = FinXClient("socket",ssl=True)
 else:
     print("set your FINX_API_KEY environment variable like os.environ[FINX_API_KEY] = my_api_key")

The FinXRest client is instantiated as follows:

Instantiate FinXRest¶
import os
import finx
from finx.finx_rest import FinXRest
finx_api_key = os.environ['FINX_API_KEY']
finx_rest_client = FinXRest(finx_api_key)

The FinxClient and FinXRest clients expose several functions:

FinXClient.calculate_security_analytics(security_id, as_of_date, price, lognormal, volatility, drift_a, yield_shift, shock_in_bp, price_as_yield, cpr, psa)¶

Return dictionary of analytics for the security submitted. If an as_of_date is entered the value is calculated as of that date with data known on that date.

Parameters:

  • secruity_id – Required Any security_identifier of any number of id schemes

  • as_of_date (str) – Optional A date formatted as “YYYY-mm-dd” (like “2022-07-01”) to use as a historical look-up of reference data.

  • price (float) – Optional A price to use for the analysis, otherwise one of [par, last best price, observation price] depending on the instrument, date selected, and API Key role

  • lognormal (binary) – Optional Whether to use “lognormal” or “normal” distribution. Default is True

  • volatility (float) – Optional The volatility of the instrument

  • drift_a (float) – Optional The mean reversion parameter for trinomial tree calculations

  • yield_shift (float) – Optional The yield shift parameter, i.e. how much the effective duration shifts for v+/v-

  • shock_in_bp (float) – Optional The basis point shock parameter, i.e. how much all rates used to calibrate eff duration get shocked in valuation

  • price_as_yield (binary) – Optional whether to use the price field entered as a Yield value. Default is False

  • cpr (float) – Optional Conditional prepayment rate. If “cpr” is entered then “psa” is ignored.

  • psa – Optional Payment speed benchmark. If “cpr” is entered then “psa” is ignored.

Returns:

List of dictionaries with cash flow details for each period forecasted.

Return type:

list

FinXClient.get_curve(curve_name, currency, start_date, end_date, country_code, fixing, tenor)¶

Return reference data, terms and conditions on a security based on its security_id. If an “as_of_date” is entered then the method returns data as it was known on that date. This can be used for retrieving historical data.

Parameters:

  • curve_name (str) – Required A curve name in the following list: []

  • currency (str) – Required Currency such as “USD”, “EUR”, “YEN”

  • start_date (str) – Required Start date of curve as string in format “YYYY-mm-dd” such as “2022-07-01”

  • end_date (str) – Optional End date of curve as string in format “YYYY-mm-dd” such as “2022-07-01”

  • country_code (str) – Optional Country code such as “USA”, “GBR”, “JPN”

  • fixing (float) – Optional Fixing rate for floating leg index

  • tenor (float) – Optional Remaining time until maturity in years

Returns:

Dictionary of key:value pairs for the security. Keys will vary based on security_type, but typical keys are [security_name, asset_class, security_type, data_source, country, currency, current_factor, maturity_date, issue_date, current_coupon, etc.]

Return type:

dict
